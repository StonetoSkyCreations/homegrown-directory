# Archive

Point-in-time reports kept for history. They were snapshots of the site at a
moment in time, not living documents, so they are parked here and excluded from
the build. Every file carries a historical-snapshot header.

The living sources of truth are, in the repo root:

- `CLAUDE.md` — operating rules for coding agents (plus `.claude/rules/`).
- `AUDIT_MASTER_PLAN.md` — the current audit baseline and staged repair plan.
  It supersedes every audit in this folder.
- `README.md` — architecture, how to add a listing, the listing-quality standard.
- `ROADMAP.md` — what is done, known issues, what is next.
- `SEO_DOCTRINE.md` — the canonical SEO rules enforced by `scripts/seo_lint.py`.
- `SOURCING_PLAN.md` + `HARVEST.md` — the NZ connection-harvest workstream.

Anything in these archived files that is still true has been folded into those
docs. Do not treat the files below as current.

- `AUDIT_REPORT.md`
- `PRELAUNCH_REPORT.md`
- `DATA_HYGIENE_REPORT.md`
- `SEO_LINT.md`
- `SEO_META_AUDIT.md`
- `SEO_STAGE_1_3_IMPLEMENTATION_REPORT.md`
- `SEO_TIGHTENING_PATCH_REPORT.md`
- `reports/` — the December 2025-era working reports (relationship audits,
  missing-entity rankings, brand umbrella audit) moved here 2026-07-02; their
  counts predate the harvest workstream (~480 listings vs 654 now).
